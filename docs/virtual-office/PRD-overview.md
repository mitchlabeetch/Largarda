# 虚拟办公室 — 产品总览

## 产品愿景

给AI Agent提供虚拟办公环境，用户以玩养成游戏的方式管理自己的AI团队。

## 核心概念

### 角色（Employee）

有名称、简介、职位的AI员工。内部由最多3个Agent实例驱动（工作/记忆/聊天），可以同时干活、整理记忆、和人聊天。

### 功能模块（Module）

赋予员工能力的工具集：

- 程序员模块：git、代码操作
- 管理模块：创建和解雇员工
- 办公软件模块：虚拟办公工具
- 其他模块：Claude tools集成

### 工作空间（Workspace）

员工自维护的个人空间，包含笔记、记忆、经验方法论、代码仓库引用。

### 模版（Template）

功能模块+角色模块的预设组合，用户可一键创建完整员工。

## 功能优先级（已更新：2026-04-01 用户最终裁决）

> **核心价值层次（用户裁决）：** 上帝视角 → 记忆持久化 → 员工协作 → 任务进度追踪 → 骨骼动画
>
> 记忆持久化是MVP命脉，不是可选功能。等级/经验/能力树永久取消，不是延后。

### P0 — 核心管理功能（MVP命脉）

- **上帝视角**：主场景一眼看清所有员工状态（颜色信号 + 当前在做什么一句话）
- **记忆持久化**：任务结束自动写记忆摘要，核心记忆注入下次任务上下文，越用越懂用户——**这是养成感的唯一真实来源**
- **员工协作沟通**：员工之间能互相发消息、接收任务、组队协作（管理模块基础）
- **任务进度追踪**：清晰知道每个任务到哪一步了（工具调用日志 + 进度状态）
- 员工创建（从模板，3种以上）
- 三实例机制（工作实例+社交实例并发）
- **骨骼动画（A档）**：PixiJS + Spine，P0必须，不接受降级

### P1 — 体验包装（糖衣层）

- 像素风房间场景渲染（工作空间视觉化）
- 程序员模块（git/代码操作）
- 搜索功能（工作空间全文搜索）
- HEALTH/STRESS/PRODUCTIVITY 颜色信号展示（不显示进度条/数值，用卡片颜色传达：绿=正常/橙=高压/红=出错/灰=空闲）
- 骨骼动画B档（退场/入场专用过渡动作）

### P2 — 扩展功能

- 日夜循环
- 办公软件模块（tools动态注入）
- 模板自定义和分享
- 跨员工搜索
- 骨骼动画C档（真实位移行走）

## 主场景（上帝视角）深化方案（2026-04-01）

### 员工卡片网格

**卡片最小信息密度（P0必须）：**

```
┌─────────────────────────────┐
│ [颜色边框]                  │
│ 小快  全栈工程师             │ ← 名字 + 职位
│ ● 工作中                    │ ← 颜色信号 + 状态文字
│ 正在修复登录bug...           │ ← 当前任务一句话（实时）
└─────────────────────────────┘
```

**颜色边框含义（P0）：**
- 绿色发光 = 工作中（STRESS ≤ 70）
- 橙色 = 高压（STRESS > 70）
- 红色 + 角标❗ = 出错
- 灰色 = 空闲
- 深灰半透明 = 休眠

**hover状态（P0）：**
- 出现快捷操作按钮：[发任务] [更多···]
- [更多···] 展开：[休眠] [进入房间] [查看记忆]

### 网格布局规则

| 员工数量 | 布局方案 | 说明 |
|---------|---------|------|
| 1-9人 | 3列网格，自适应行数 | 一屏或轻微滚动 |
| 10-49人 | 按状态分组 + 虚拟滚动 | 出错组置顶，其余按状态排列 |
| 50人以上 | 搜索优先 + 分组折叠 | 默认只展开出错组 |

**分组排序（固定优先级）：**
1. 出错（红色）— 始终最顶，始终展开
2. 高压（橙色）— 次优先
3. 工作中（绿色）
4. 空闲（灰色）
5. 休眠（深灰）

### 状态实时推送

**产品要求（架构实现）：**
- 状态变化延迟 < 2秒到达主场景
- 卡片状态文字实时更新（打字机效果流式显示）
- 从"出错"状态变化时立即推送，不等2秒批处理

**推送事件类型（产品定义，架构决定实现方式）：**

| 事件 | 触发时机 | 更新内容 |
|------|---------|---------|
| 状态变化 | 工作中/空闲/出错切换 | 边框颜色 + 状态文字 |
| 任务进度 | 工具调用完成 | 当前任务一句话描述 |
| 出错 | TaskStatus: failed | 红色边框 + 角标 + Toast |
| 任务完成 | TaskStatus: completed | 颜色恢复空闲 + 庆祝动效 |

**性能约束：**
- 50人以上时，不在可视区域外的卡片不渲染动画（IntersectionObserver）
- 状态推送批处理：同一秒内的多个更新合并为一次渲染
- 虚拟滚动：只渲染可视区域 ±1屏的卡片

### 验收标准（主场景P0）

- [ ] 打开主场景，0步看到所有员工状态（颜色 + 一句话）
- [ ] 员工出错时，< 2秒在主场景看到红色边框 + 角标
- [ ] hover员工卡片，出现[发任务]按钮，点击弹出输入框，不需要进入工作空间
- [ ] 50人以下性能流畅（60fps），50人以上虚拟滚动无卡顿
