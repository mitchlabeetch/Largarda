## 一、技术方向 A：图转动画

### 1.1 Animated Drawings（Meta）

**状态：已归档（2025-09-03），不再维护**

作者在 README 中宣布："I'm moving on to new adventures and won't be able to maintain this repository anymore, so I've chosen to archive it."

- **原理：** 人形姿态估计 + 骨骼自动绑定，将手绘人物图片驱动为动画
- **输出格式：** MP4、透明 GIF（有导出透明 GIF 的示例配置）
- **像素风适配：** 差。算法依赖人形轮廓检测，像素角色（16×32px 基础网格）轮廓粗糙，检测成功率低；且工具面向"儿童涂鸦风格"插画，不是像素艺术
- **循环动画：** 支持，但需要手动指定动作 YAML 配置，没有状态枚举驱动
- **成本：** 免费（Python 本地跑）
- **已归档风险：** Python 3.8 依赖，新环境安装可能失败；无人修 bug

**结论：不推荐。工具已死，且不适合像素风。**

---

### 1.2 Live2D

- **原理：** 将静态插画分层（头发、眼睛、身体各层），通过变形网格实现骨骼驱动动画
- **像素风适配：** 差。Live2D 设计目标是高精度插画（日式 ACG 风格），像素艺术的锯齿感会被变形网格破坏，放大后锯齿丢失
- **编辑器：** Cubism Editor，个人免费版（1080p 以下输出），商业版 $10/月起
- **运行时：** `live2d-core`，npm 可用，有 PixiJS 集成示例
- **透明背景：** 支持
- **成本：** 个人免费版够 MVP，商业版按月收费

**结论：不推荐。技术上能用，但像素风格与 Live2D 审美方向冲突，变形网格会破坏像素锯齿感。**

---

### 1.3 Spine / DragonBones 自动骨骼绑定工具

- **Spine 自动绑定（Spine 4.x Mesh 功能）：** Spine Editor 可以半自动切割 spritesheet 并生成骨骼，但仍需 $69 编辑器授权，预算已否决
- **DragonBones 自动绑定：** DragonBones Pro 已停更（2021），自动绑定功能随工具失效
- **第三方 Auto-Rig 工具（如 Mixamo）：** 面向 3D 模型，不适用 2D 像素艺术

**结论：无可用工具，不适用。**

---

## 二、技术方向 B：AI 视频生成

### 共性问题（对所有 AI 视频工具适用）

1. **透明背景：** 所有主流 AI 视频工具默认输出不透明背景，需要 rembg/chroma-key 后处理抠图。像素艺术抠图容易误删像素边缘的半透明像素，导致锯齿消失、风格破坏
2. **像素风格一致性：** 视频模型内部是连续帧，像素艺术的离散锯齿感很难在帧间保持一致，帧与帧之间会有"风格抖动"
3. **循环动画：** 绝大多数工具不支持生成完美首尾对接的循环视频，需要手动对接末帧/首帧

---

### 2.1 Runway Gen-3 / Gen-4

- **功能：** 图像驱动视频生成（Image-to-Video），输入参考图 + 文字描述 → 输出 5-10 秒视频
- **像素风适配：** 一般。Gen-4 支持"风格参考图"功能，理论上可以锁定像素风格，但帧间一致性无法保证
- **透明背景：** 不支持，需后处理（rembg + 阈值抠图）
- **循环动画：** 不支持自动循环，需手动剪辑
- **成本：** 免费版每月 125 credits（约生成 3-5 段视频），Standard $12/月，Unlimited $28/月
- **输出格式：** MP4，最大 768×768，无 WebM/透明格式

**结论：可以作为"美术验证工具"（快速看效果），不推荐作为最终动画资产生产方案。**

---

### 2.2 Pika Labs

- **功能：** 文字/图片驱动视频生成，擅长短时循环动画（Pikaffects）
- **像素风适配：** Pika 对像素风理解尚可（社区有案例），但帧间风格抖动仍存在
- **透明背景：** 不支持（与 Runway 同等限制）
- **循环动画：** 有 "seamless loop" 模式，效果不稳定
- **成本：** 免费版每月 150 credits，Basic $8/月
- **适用场景：** 快速原型验证（看"idle 呼吸感"是否可行），不作为生产资产

**结论：与 Runway 同级，可作为原型验证，不作为生产方案。**

---

### 2.3 Kling AI

- **功能：** 图转视频，效果接近 Runway Gen-3，中国产品
- **透明背景：** 不支持
- **循环动画：** 不支持自动循环
- **成本：** 有免费额度
- **结论：与 Runway/Pika 等级相当，无差异化优势，不做单独推荐。**

---

### 2.4 Stable Video Diffusion（SVD，本地部署）

- **仓库：** `Stability-AI/generative-models`（2025-12 仍有更新，stars 27k）
- **原理：** 图片 → 25 帧短视频，支持本地 GPU 运行
- **像素风适配：** 中等。SVD 是通用视频生成，不专门优化像素艺术；同样存在帧间风格抖动
- **透明背景：** 不支持（RGBA 输出需配合 `rembg` 后处理）
- **循环动画：** 不支持自动首尾对接，需 `ffmpeg` 剪辑
- **硬件要求：** 最低 16GB VRAM（NVIDIA RTX 3080 以上），无 GPU 无法本地跑
- **成本：** 模型免费，但需要 GPU 服务器（Vast.ai 约 $0.3-0.8/小时）

**结论：本地跑成本取决于 GPU。无 GPU 则不可用；有 GPU 可作为序列帧辅助生产工具（生成草稿帧再手工精修），不作为直接资产。**

## 三、技术方向 C：程序化动画

### 3.1 Rive（骨骼动画，推荐方案）

- **编辑器：** Rive Editor（Web 版，免费）
- **运行时：** `@rive-app/canvas` 或 `@rive-app/webgl2`（MIT 开源，**4 天前发版 v2.36.0**，活跃维护）
- **像素风适配：** 中等偏好。Rive 是矢量骨骼工具，像素艺术需要以 spritesheet 纹理贴图替换矢量图形；Canvas 渲染时可设置 `image-rendering: pixelated` 保持锯齿感
- **透明背景：** 支持（Canvas 背景透明）
- **循环动画：** 原生支持，State Machine 可定义 idle/working 等状态枚举及切换条件
- **骨骼换装：** 支持（Slot 替换），一套骨架多职位换装可行
- **与 PixiJS 集成：** 需要自建桥接层——Rive 渲染到独立 `<canvas>`，PixiJS 通过 `PIXI.Texture.from(canvas)` 将 Rive canvas 作为纹理读取并渲染到 PixiJS stage
- **成本：** 编辑器完全免费，运行时 MIT

**Rive + PixiJS 桥接方案（伪代码）：**

```typescript
// Rive canvas 作为 PixiJS 动态纹理
const riveCanvas = document.createElement('canvas');
const rive = new Rive({ canvas: riveCanvas, src: 'employee.riv', autoplay: true });

// PixiJS 每帧从 riveCanvas 更新纹理
const texture = PIXI.Texture.from(riveCanvas);
const sprite = new PIXI.Sprite(texture);
app.ticker.add(() => { texture.update(); });

// 状态切换
rive.stateMachineInputs('State Machine 1')
  .find(i => i.name === 'state')
  .value = 'working';
```

**风险：**
- Rive canvas 的像素采样率需要手动设置（`devicePixelRatio` 倍数），否则 PixiJS 纹理读取时会出现模糊
- State Machine 需要在 Rive Editor 中定义，美术需要学习 Rive 工作流（学习曲线约 1-2 天）
- 桥接层每帧 `texture.update()` 有性能开销，多角色时需要评估帧率

**结论：推荐。免费、活跃维护、支持骨骼换装和状态机，是 Spine 最接近的免费替代。**

---

### 3.2 PixiJS 序列帧（PIXI.AnimatedSprite，当前技术路径延续）

- **原理：** 不引入骨骼系统，直接用 PixiJS 内置 `AnimatedSprite` 播放 spritesheet 序列帧
- **像素风适配：** 原生最佳（`SCALE_MODES.NEAREST`，天然保留锯齿）
- **透明背景：** 支持（PNG spritesheet 透明通道）
- **循环动画：** 原生支持，`loop = true/false`
- **骨骼换装：** 不支持，每个职位/状态需要独立绘制序列帧
- **美术工具：** Aseprite（$20）或免费替代 LibreSprite / Pixelorama（完全免费）
- **AI 加速帧生成：** 可用 PixelLab（AI 像素艺术工具）生成草稿帧，再人工精修；约 1-2 天/职位
- **成本：** $0（用 LibreSprite）或 $20（Aseprite）

**每职位工作量估算（75 帧/职位，7 个状态）：**

| 方式 | 工时/职位 | 质量 |
|------|---------|------|
| 纯手绘（Aseprite）| 5-7 天 | 最高 |
| AI 草稿 + 人工精修 | 2-3 天 | 高 |
| LPC 开源素材改绘 | 1-2 天 | 中（风格接近但不完全一致）|

**结论：技术上最简单、稳定性最高，是当前老锤预览器已验证的路径。代价是每职位需要单独制作所有帧，无骨骼复用优势。**

---

### 3.3 CSS Sprite 动画

- 比 PixiJS 序列帧更轻量，适合主场景卡片内 48×48px 的小角色动画
- 不适合工作空间内的大尺寸骨骼动画
- 作为降级方案（PixiJS 不可用时）保留

## 六、推荐路径

### MVP 推荐：序列帧（最小风险）

**理由：** 老锤的 PixiJS 预览器已通过测试，序列帧是 PixiJS 内置方案，无额外依赖，技术路径完全打通。

步骤：
1. 美术用 LibreSprite（免费）或 Aseprite（$20）制作 3 职位 × 7 状态序列帧（约 75 帧/职位）
2. 导出透明 PNG spritesheet
3. PixiJS `AnimatedSprite` + `SCALE_MODES.NEAREST` 直接播放
4. 状态切换逻辑手写（`setCurrentFrame(0); play()`）

### 长期推荐：Rive（骨骼换装，最接近 Spine 的免费替代）

**理由：** 一套骨架支持多职位换装，减少美术长期维护成本。Rive Editor 免费，运行时 MIT，工具活跃。

步骤：
1. 美术在 Rive Editor 中制作骨骼角色，配置 State Machine（idle/working/chatting 等状态）
2. 多职位通过 Slot 换装实现
3. 开发在 PixiJS 中实现 Rive canvas → PixiJS texture 桥接层
4. 同样用 `SCALE_MODES.NEAREST` 保持像素锯齿感

### AI 视频工具的定位：原型验证专用，不作为生产资产

可以用 Runway/Pika 快速验证"idle 呼吸感""working 打字感"的效果方向，给美术参考。但最终资产必须走序列帧或 Rive 路径，不用 AI 视频直接作为动画资产。

## 七、结论与建议

**阿构的建议：**

1. **立即可行（MVP 阶段）：序列帧路径**。预览器已通过测试，美术按 design-final.md §12 的帧数规格（75帧/职位）制作，技术侧零变动。工具用 LibreSprite（免费）。
2. **中期切换（P1）：Rive 路径**。一旦美术完成 1 套 Rive 骨骼验证（约 2 天），评估桥接层性能，确认效果后批量迁移。
3. **AI 视频工具：不进入技术选型**，仅供美术参考效果方向用。

**需要用户裁决的问题：**
- 是接受序列帧路径（美术工作量更大，但技术最简单）？
- 还是要求先做 Rive 方案验证（骨骼换装优势，但有额外集成工作）？
