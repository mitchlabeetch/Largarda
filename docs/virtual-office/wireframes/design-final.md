## 二、主场景布局（上帝视角）

> **优先级说明（2026-04-01 确认）：** 主场景的信息密度和可读性优先于房间动画精度。
> 用户核心需求是"0.5秒内一眼看清所有员工状态"，像素动画是糖衣，不是主菜。
>
> **布局决策：** 主场景 = 员工卡片网格（高信息密度）。横向房间预览视图不在主场景出现，
> 仅在点击员工卡片后进入工作空间时才渲染。

### 整体结构：1440px 宽，员工数量不限，卡片网格支持滚动

> **2026-04-01 裁决：** 员工数量无上限。主场景网格超出视口高度时整体可滚动，
> 使用虚拟列表（react-virtual 或 FixedSizeGrid）保证大量员工时不卡顿。
> 卡片列数固定3列，行数按员工数量自动增长，末尾始终有一个「+ 招募成员」空卡位。

```
┌─────────────────────────────────────────────────────────────────────────────────┐

├─────────────────────────────────────────────────────────────────────────────────┤

│   ┌────────────────────┐  ┌────────────────────┐  ┌────────────────────┐        │

│   │  [工作中，绿色边框] │  │  [空闲，正常边框]   │  │  [工作中，绿色边框] │        │
│   └────────────────────┘  └────────────────────┘  └────────────────────┘        │
```

**虚拟列表方案（性能要求）：**

- 员工数 ≤ 20：正常渲染，无需虚拟化
- 员工数 > 20：启用 `react-virtual` FixedSizeGrid，只渲染可视区域行
- 卡片高度固定 220px（标准）/ 280px（hover 展开），网格 gap 24px
- 滚动时固定顶部导航栏（`position: sticky; top: 0; z-index: 100`）

### 员工卡片规格（最终版）

尺寸：160×200px（标准）| hover 展开：200×260px（增加底部操作区）

> **2026-04-01 裁决：** 主场景卡片不显示 HEALTH/STRESS/PRODUCTIVITY 进度条数字。
> 状态信息通过卡片边框颜色 + 状态指示点颜色传达，不放进度条。

**设计决策：状态用颜色信号传达，不用进度条数字**

用户裁决（2026-04-01 第二批）：主场景卡片不显示 HEALTH/STRESS/PRODUCTIVITY 进度条。
状态通过以下视觉信号传达，用户 0.5 秒内一眼判断：

| 状态   | 卡片边框颜色                | 指示点             | 卡片亮度            |
| ------ | --------------------------- | ------------------ | ------------------- |
| 工作中 | `--vo-state-working` 暖琥珀 | ● 暖琥珀，pulse    | 100%                |
| 空闲   | `--vo-border-card` 正常     | ○ 灰色，静止       | 100%                |
| 高压   | `--vo-state-stress` 橙色    | ● 橙色，快速 pulse | 100%                |
| 出错   | `--vo-state-error` 红色     | ● 红色，闪烁       | 100%                |
| 休眠   | 无                          | 无                 | 20%（整卡片降亮度） |

## 三、员工工作空间（像素房间 + 信息抽屉）

### 进入方式：点击卡片 → 400ms 转场 → 进入房间聚焦视图

**转场序列（400ms）：**

1. 卡片 scale 1.0 → 1.1，周边卡片渐隐（100ms）
2. 摄像机推近（zoom + pan），房间展开填满屏幕（200ms）
3. 顶部状态栏从顶部滑入，底部 HUD 从底部滑入（100ms）

**重要：** 转场动画期间底部 HUD 输入栏已可用，不等动画结束。

## 四、角色动画规范（骨骼动画版，2026-04-01 确认）

> **技术方案确认（MVP P0）：** PixiJS + DragonBones（免费开源，编辑器 + pixi-dragonbones 运行时）。
> 不需要寻路，不需要 A\* 寻路，核心是：一套骨架 → 多个动作 → 状态枚举切换。
> 每个状态必须有生动循环动画，不允许静态图，角色必须"活的"。
> 详细工具链评估见 Section 12。

### 像素角色规格

- 基础网格：16×32px（横版侧视，高于宽），渲染为 48×96px（3x scale）
- 骨骼动画在 PixiJS Canvas 层渲染，DOM 层叠加 React UI

```css
/* Canvas 内像素画渲染质量保证（PixiJS 配置） */
/* app = new PIXI.Application({ ... }) */
/* PIXI.settings.SCALE_MODE = PIXI.SCALE_MODES.NEAREST; */
/* 等价 CSS：image-rendering: pixelated（仅 fallback 用，骨骼动画走 Canvas）*/
.vo-pixel-avatar--fallback {
  image-rendering: pixelated; /* Chrome / Safari / Edge */
  image-rendering: crisp-edges; /* Firefox */
}
```

### 状态定义（枚举，不需要寻路）

| 状态         | 触发条件         | 骨骼动作名   | 位置         | 含义                              |
| ------------ | ---------------- | ------------ | ------------ | --------------------------------- |
| `idle`       | 无任务           | `anim_idle`  | 桌前椅上     | 站立待机，轻微呼吸                |
| `working`    | 工作实例活跃     | `anim_work`  | 桌前椅上     | 坐姿打字，看屏幕                  |
| `chatting`   | 聊天实例收到消息 | `anim_chat`  | 桌前椅上     | 面向用户，嘴部/手势               |
| `memorizing` | 记忆实例活跃     | `anim_read`  | 书架旁       | 书架旁翻书整理                    |
| `resting`    | 完全空闲         | `anim_rest`  | 房间中央偏右 | 伸懒腰/环顾/发呆随机轮换          |
| `sleeping`   | 用户手动休眠     | `anim_sleep` | 床边         | 呼吸起伏循环，色调变暗            |
| `happy`      | 任务完成         | `anim_happy` | 桌前椅上     | 庆祝跳跃（单次）                  |
| `error`      | 任务失败         | `anim_error` | 桌前椅上     | 叹气摊手，等待处理                |

> **resting / sleeping 语义说明（郭聪明确认，2026-04-01）：**
> `resting` = 完全空闲时的自然休息状态（无任务，在房间里放松）；
> `sleeping` = 用户手动触发的休眠（接受新任务关闭）。
> 两者骨骼动作可复用同一套（`idle` + `resting` 动作组共用），资产制作不需要做两套。

### 动画分级规范（A/B/C 三级）

**A 级 — 状态循环动画（MVP P0，骨骼动画实现）**

每个状态有生动循环骨骼动画，角色在当前位置持续播放，不允许静止：

| 状态       | 骨骼动作     | 帧率  | 循环       | 微变化要求                                   |
| ---------- | ------------ | ----- | ---------- | -------------------------------------------- |
| idle       | `anim_idle`  | 6fps  | 是         | 呼吸浮动 + 眨眼 + 头轻转，每8帧一循环        |
| working    | `anim_work`  | 10fps | 是         | 手部打字节奏 + 抬头看屏幕 + 低头继续，每10帧 |
| chatting   | `anim_chat`  | 8fps  | 是         | 嘴部开合 + 手势抬起放下，每8帧               |
| memorizing | `anim_read`  | 7fps  | 是         | 翻页 + 托腮 + 手指划过，每10帧               |
| resting    | `anim_rest`  | 5fps  | 是         | 伸懒腰/环顾/发呆随机轮换，每8帧（可复用 idle 动作组）|
| sleeping   | `anim_sleep` | 3fps  | 是         | 胸口极慢起伏 + 色调变暗，接近静止但不是静止，每8帧 |
| happy      | `anim_happy` | 12fps | 否（单次） | 跳跃庆祝，播完回 idle                        |
| error      | `anim_error` | 5fps  | 是         | 停下+叹气+摊手困惑，每8帧                    |

**B 级 — 状态切换过渡（MVP 必须，核心丝滑感来源）**

状态切换时播放退出帧 → 淡出 → 淡入 → 进入帧，不是瞬间切换：

```
退出当前状态（exit frames）：2-3帧，200ms
  淡出（opacity 1→0）：150ms，ease-out
  角色位置瞬移到目标位置（不走路，直接到位）
  淡入（opacity 0→1）：150ms，ease-in
进入新状态（enter frames）：2-3帧，200ms

总过渡时长：约 700ms
```

各切换方向的过渡帧：

| 切换               | 退出帧          | 进入帧           |
| ------------------ | --------------- | ---------------- |
| idle → working     | 走向桌子（2帧） | 坐下（2帧）      |
| working → idle     | 起身（2帧）     | 站立放松（2帧）  |
| working → chatting | 转身（2帧）     | 招手/开口（2帧） |
| any → happy        | 停止当前（1帧） | 跳起（2帧）      |
| any → sleep        | 走向床（2帧）   | 躺下（2帧）      |
| sleep → any        | 起身（2帧）     | 伸懒腰（2帧）    |

**C 级 — 增强细节（P1，非 MVP）**

- 随机 idle 变体：每30-60秒触发一次特殊 idle（喝咖啡/看窗外/伸展）
- 压力动作：STRESS > 70% 时打字动作变快，偶发揉眼睛动作
- 完成连击：连续完成任务时 happy 动画升级（更多粒子）

### 骨骼动画状态切换实现（PixiJS + DragonBones）

**PixiJS 层骨骼动画切换伪代码：**

```typescript
// 状态切换函数（骨骼动画版）
function switchEmployeeState(
  armature: dragonBones.Armature, // DragonBones 骨骼对象
  fromState: EmployeeState,
  toState: EmployeeState,
  pixiApp: PIXI.Application
) {
  // 1. 退出动画：当前骨骼动作渐出
  pixiApp.ticker.add(fadeOutTicker); // opacity 1→0，150ms

  setTimeout(() => {
    // 2. 切换骨骼动作（DragonBones API）
    // armature.animation.play('anim_work', 0)  // 0 = 无限循环
    armature.animation.play(STATE_ANIM_MAP[toState], toState !== 'happy' ? 0 : 1);

    // 3. 进入动画：新状态骨骼动作渐入
    pixiApp.ticker.add(fadeInTicker); // opacity 0→1，150ms + translateY(4→0)
  }, 150);
}

// 状态到骨骼动作名映射
const STATE_ANIM_MAP: Record<EmployeeState, string> = {
  idle: 'anim_idle',
  working: 'anim_work',
  chatting: 'anim_chat',
  memorizing: 'anim_read',
  resting: 'anim_rest',
  sleeping: 'anim_sleep',
  happy: 'anim_happy', // loop=false，播完自动回 idle
  error: 'anim_error',
};
```

**状态专属视觉叠加（DOM/Canvas 双层）：**

```css
/* working 状态：Canvas 层电脑屏幕青光投射到角色（PixiJS ColorMatrixFilter）*/
/* CSS 层兜底（骨骼渲染在 Canvas，此为降级）：*/
.vo-sprite-container--working {
  filter: drop-shadow(0 0 4px rgba(0, 255, 204, 0.3));
}

/* sleep 状态：低透明 + 灰度（PixiJS alpha + ColorMatrixFilter.desaturate()）*/
.vo-sprite-container--sleep {
  opacity: 0.35;
  filter: grayscale(0.8);
}

/* prefers-reduced-motion：跳过过渡，直接切换骨骼动作 */
@media (prefers-reduced-motion: reduce) {
  /* PixiJS 层：跳过 fadeOut/fadeIn ticker，直接 setAnimation */
  /* DOM 层降级 fallback：*/
  .vo-sprite-container {
    transition: none !important;
    animation: none !important;
  }
}
```

## 十二、序列帧动画可行性评估

> 视角确认：**横版侧视（side-scrolling）**，参考图视角（类似星露谷/lofi girl）。
> 动画已简化为状态枚举 + 淡入淡出过渡，**无寻路、无行走动画**。
> 角色位置固定，状态切换时 fade out → 位置瞬移 → fade in，不需要行走序列帧。
>
> **用户底线（2026-04-01 确认）：每个状态必须有生动循环动画，不是静态图。**
> 切换可以简化，但角色必须"活的"——每个状态 8-12 帧循环，有微变化（呼吸/手部/眼神）。

### 各动作帧数规格（用户确认版）

| 动作                        | 帧数 | 帧率  | 播放模式 | 微变化细节                                                                   |
| --------------------------- | ---- | ----- | -------- | ---------------------------------------------------------------------------- |
| 站立待机（idle）            | 8帧  | 6fps  | 循环     | 帧1-4：轻微上下浮动（呼吸感）；帧5：眨眼；帧6-7：头轻微转向；帧8：回正       |
| 打字工作（working）         | 10帧 | 10fps | 循环     | 帧1-6：双手交替敲键盘节奏；帧7-8：抬头看屏幕（眼神移动）；帧9-10：低头继续打 |
| 聊天（chatting）            | 8帧  | 8fps  | 循环     | 帧1-4：嘴部开合（说话感）；帧5-6：手势抬起；帧7-8：回到放松姿势              |
| 翻书/整理记忆（memorizing） | 10帧 | 7fps  | 循环     | 帧1-4：手翻书页动作；帧5-6：托腮思考；帧7-9：手指划过文字；帧10：翻下一页    |
| 空闲休息（resting）         | 8帧  | 5fps  | 循环     | 帧1-3：坐着伸懒腰（双臂上举）；帧4：放下；帧5-7：发呆（眼神游离）；帧8：回神（可复用 idle 动作组）|
| 休眠（sleeping）            | 8帧  | 3fps  | 循环     | 帧1-4：胸口缓慢上起（吸气）；帧5-8：缓慢下落（呼气）；极慢接近静止；色调变暗 |
| 出错等待（error）           | 8帧  | 5fps  | 循环     | 帧1-3：停下，看屏幕；帧4-5：叹气（肩膀下沉）；帧6-7：摊手困惑；帧8：定格等待 |
| 出入场过渡（B档）           | 4帧  | 8fps  | 单次     | 坐下/起身动作，B档才需要，可从 idle/working 各取2帧拼接                      |
| 完成庆祝（happy）           | 10帧 | 12fps | 单次     | 帧1-4：跳起；帧5：空中定格；帧6-8：落地；帧9-10：举手庆祝；粒子由代码生成    |
| 静态立绘（主场景卡片用）    | 1帧  | -     | 静态     | 卡片内 48×48px 立绘，不需要动画                                              |

**单角色有效帧数：约 75帧（MVP A档，6个循环状态 + 庆祝）**
（不含 B档过渡帧；行走序列帧已取消）

**循环末帧对接要求（硬性）：** 每个循环动画末帧必须能自然衔接首帧，无跳帧感。
制作时以首帧为参照，末帧姿势与首帧误差 ≤ 1px。

### 生成方式

> **用户裁决（2026-04-01，已锁定）：** 全部自绘，不使用开源素材，不购买素材包。

| 方式                          | 时间/职位  | 质量             | 备注                                      |
| ----------------------------- | ---------- | ---------------- | ----------------------------------------- |
| 手绘（Aseprite）              | 5-7天/职位 | 最高             | 正式资产路线，符合裁决                    |
| AI辅助（PixelLab + 人工校正） | 2-3天/职位 | 中高             | 辅助加速，仍需人工定稿，符合自绘精神      |
| 参考图直接拆分                | 不可行     | -                | 参考图角色为静态，无法提取序列帧          |

MVP 1个职位先出，Aseprite 手绘约5-7天，含每状态8-12帧。

### 职位外观差异方案

- 75帧骨架所有职位共用（MVP A档）
- 服装/发色通过 Aseprite 调色板替换，约1-2小时/职位

## 十三、给开发-小快的协作备注

- 色彩变量（`--vo-state-*`）直接对应流程状态机节点，每个状态都有对应视觉
- 角色坐标系需要由 Canvas 层持久化（x, y），DOM 层的右侧抽屉位置由此坐标决定

- 顶部状态栏格式（裁决更新，2026-04-01）：`TIME: HH:MM PM | STATUS: WORKING | 当前任务一句话`，无 H/S/P 进度条，像素风分隔符
- 家具位置参考图已确认：床在左，工作区（电脑+书架+台灯）在右，窗户右上角，植物窗边，角色坐于桌前中右区域
