## 二、主场景布局（横版侧视像素房间）

### 视图类型：横版侧视（side-scrolling），非俯视 RPG

参考图已确认：暖棕木质色调的横版房间，角色坐在电脑桌前工作。每个员工拥有独立的像素风格房间，玩家可以横向滚动浏览不同员工的房间。

### 渲染层架构（5层，由底至顶）

```
Layer 5 (z=50):  React DOM UI 层      — 员工信息条、状态 badge、工具弹窗
Layer 4 (z=40):  角色 Sprite 层       — 骨骼动画角色，含阴影
Layer 3 (z=30):  家具/道具 Sprite 层  — 书桌、电脑、床、书架，可与角色互动
Layer 2 (z=20):  昼夜 CSS 叠加层      — mix-blend-mode:multiply，颜色温度变化
Layer 1 (z=10):  背景 PNG 层          — 预渲染的房间背景（墙、地板、窗户）
```

实现技术：Phaser（推荐，原生支持 Tilemap + A\* 寻路 + Sprite Sheet）
像素缩放：原始分辨率 ~320×180px，以 3x 或 4x 渲染到屏幕（1440×900 桌面端裁剪可视区域）

## 四、像素角色设计规范

### 像素网格：16×16 基础，渲染为 48×48 CSS px（3x 放大）

### image-rendering 必须设置（防止浏览器抗锯齿模糊像素图）

```css
.vo-sprite,
.vo-pixel-avatar,
.vo-pixel-icon {
  image-rendering: pixelated; /* Chrome / Safari / Edge */
  image-rendering: crisp-edges; /* Firefox */
}
```

**这不是可选项，是强制规范。** 不设置时 Retina 屏（devicePixelRatio ≥ 2）上像素画会双线性插值变模糊，直接毁掉像素风格。

## 六、状态动效规范

### 角色动画状态机（房间场景）

```
状态节点：
  idle_stand      — 站立待机，轻微上下浮动，偶尔眨眼（8帧循环）
  idle_walk_E     — 向右走（6帧循环）
  idle_walk_W     — 向左走（6帧循环）
  sit_down        — 坐下动作（4帧，单次播放）
  stand_up        — 起身动作（4帧，单次播放）
  working         — 坐姿打字（12帧循环，手部往复）
  resting_bed     — 床上躺着（4帧循环，胸口起伏）
  happy           — 跳跃+星星粒子（10帧，单次播放）
  sleep           — 关灯，角色静止低透明度

状态转换触发条件：
  空闲 → 工作中：工作实例收到新任务 → sit_down → working
  工作中 → 完成：任务结束 → happy → stand_up → idle_stand
  工作中 → 整理记忆：记忆实例激活 → 角色走到书架 → idle_walk_W/E → idle_stand（书架旁）
  任意 → 对话：聊天实例收到消息 → 角色转向屏幕（朝玩家） → 头顶「...」气泡
  工作中 → 休眠：空闲超时 → stand_up → idle_walk → resting_bed → sleep
  休眠 → 唤醒：任务来临 → 角色起身（stand_up）→ 走向电脑 → sit_down → working
```

### 休眠 → 唤醒序列（房间场景版）

```
1. 房间灯光从暗淡恢复（亮度 30% → 100%，400ms overlay 淡出）
2. 角色在床上起身动画（stand_up，4帧，400ms）
3. 角色步行到电脑桌（idle_walk，路径由 A* 计算，约 800ms）

5. 电脑屏幕亮起（屏幕从黑色 → 发光，200ms）
6. 状态条 STATUS 由 SLEEPING → WORKING（即时）
总时长：约 1900ms（全序列自然播放，不可跳过）
```

### 任务完成动效

```
1. 角色播放 happy 动画（跳跃，10帧，600ms）
2. 8颗 --vo-warm-lamp (#F5A623) 星星粒子从角色头顶散射消失（800ms）
3. 状态条数值更新：EXP +N（数字从状态条飘出向上淡出，500ms）
4. 若 EXP 满：升级粒子效果（金色粒子爆发，Level 数字变化，1000ms）
```

```

2. 玩家点击创建 → 全屏覆盖层（见第九节）

```

### 多 Agent 协作的视觉连线（基于 AgentTool）

AgentTool 触发子 Agent 时，员工卡片之间出现连线动画：

```
┌──────────┐         ┌──────────┐
│  小快     │  ══════►│  阿构     │   ← 调用关系连线
│ Frontend  │         │ Backend   │     颜色：调用者的职位色
│  ●工作中  │         │  ●工作中  │
└──────────┘         └──────────┘
      ↑ 发出子任务：「设计接口文档」
      连线：pulse 动画，从左到右粒子流动
      颜色：--vo-primary（活跃）
```

连线规范：

- 颜色：调用方的职位颜色
- 粗细：1-2px，沿直线或 L 型路径
- 动画：3-5px 光点从起点流向终点，间隔 600ms，duration 800ms
- 消失：子 Agent 完成后，连线淡出（300ms fade）
- 层叠：最多同时显示 3 条连线，超出则堆叠显示数字角标
